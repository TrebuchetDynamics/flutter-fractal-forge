## Critical Issue: Nova Fractal Uniform Mismatch

**Severity:** CRITICAL  
**Impact:** Nova fractal renders with corrupted parameters  
**Files Affected:** 
- `lib/core/modules/nova_module.dart`
- `shaders/nova_gpu.frag`

### The Problem

The Dart module binds uniforms in this order:
```
[0] time
[1] size.width
[2] size.height
[3] pan.x
[4] pan.y
[5] zoom
[6] iterations
[7] relaxation
[8] colorScheme
[9] transparentBg
```

But the shader expects:
```
[0] uTime
[1] uResolution (vec2 - takes 2 slots)
[2] uResolution
[3] uCenter (vec2 - takes 2 slots)
[4] uCenter
[4] uIterations
[5] uRelaxation
[6] uColorScheme
[7] uTransparentBg
```

### Impact Analysis

Due to this mismatch:
- `uIterations` receives `pan.y` instead of iteration count
- `uRelaxation` receives `zoom` instead of relaxation parameter
- `uColorScheme` receives `iterations` instead of color scheme
- `uTransparentBg` receives `relaxation` instead of transparency flag

This causes the Nova fractal to render incorrectly with wrong iteration counts, zoom behavior, and colors.

### Root Cause

The module's `setUniforms` callback uses separate `setFloat` calls for vec2 components (width, height, pan.x, pan.y), but counts them as if they were single uniforms. This creates an index offset starting at position 1.
