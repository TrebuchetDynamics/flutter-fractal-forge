================================================================================
CRITICAL IMPLEMENTATION PATTERNS
================================================================================

1. LOCATION SPECIFICATION (FractaVista model)
   JSON format with: centerX, centerY, zoom, maxIterations, escapeRadius
   File extension: .fracta
   Serialization: nlohmann/json C++ library

2. GPU COMPUTE DISPATCH (FractaVista model)
   • Workgroup size: 16×16
   • Calculation: (width+15)/16 × (height+15)/16 groups
   • Memory barrier: GL_SHADER_IMAGE_ACCESS_BARRIER_BIT
   • Output binding: rgba8 image at binding 0

3. SUPERSAMPLING (FractaVista model)
   • Dispatch at supersampled resolution (2×, 4×, 8×)
   • Read back with glGetTexImage()
   • Flip y-axis (OpenGL convention)
   • Export via SDL_image or equivalent

4. SHADER COMPILATION (FractaVista model)
   • Load shader source from file
   • Inject #define directives after #version line
   • Compile GL_COMPUTE_SHADER
   • Link program and bind UBO

5. PRECISION SELECTION (Fractl model)
   • Compile-time feature flags (cfg_if)
   • Mutually exclusive constraints
   • Runtime dispatch to selected implementation

6. PALETTE INTERPOLATION (FractaVista model)
   • ColorStop[] array with position field
   • Binary search for active range
   • Linear interpolation between stops
   • Smoothing frequency parameter for transitions

7. MAIN BULB OPTIMIZATION (All projects)
   • Pre-compute: q = (x - 0.25)² + y²
   • Check: q * (q + (x - 0.25)) ≤ 0.25 * y²
   • If true: return maxIterations immediately

8. SMOOTH COLORING (DeepDrill, FractaVista)
   • Normalized Iteration Count (NIC)
   • Formula: iterations + 1 - log(log(|z|)) / log(2)
   • Fractional part enables smooth color transitions

9. VIDEO EXPORT (DeepMake model)
   • INI config → parameter parser
   • Compute frames via renderer
   • Pipe frames to FFmpeg via named pipe
   • FFmpeg encodes to video (H.264/VP9)

10. REFERENCE CACHING (DeepDrill model)
    • Compute reference point at high precision
    • Reuse for multiple nearby pixels (perturbation)
    • Detect glitches, pick new reference if needed
    • Multi-round refinement for accuracy
