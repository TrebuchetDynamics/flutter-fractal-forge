# Fractal Pipeline — Stage E: Maintenance & Dashboard Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- []`) syntax for tracking.

**Goal:** Implement 6 maintenance jobs + static HTML dashboard that keep a growing catalog coherent. Link rot, taxonomy drift, shader regression, duplicate re-checks, citation refresh, all automated via cron. Dashboard is static HTML regenerated by any maintenance run — no backend needed.

**Architecture:** All jobs live under `scripts/research/maintenance/`, invoked via `forge maintain <job>`. Idempotent. Log to `reports/logs/`. Open PRs rather than commit directly when they find fixes. Dashboard reads `fractal_registry.yaml` + quality blocks, renders `reports/dashboard.html`. Cron schedules in `.github/workflows/maintenance-cron.yml`.

**Tech Stack:** Python 3.11 (existing), **NEW**: `jinja2` (already from Stage B, reused for HTML), optionally `plotly==5.24.1` for interactive charts in dashboard.

**Prerequisites:** Stages A + B done. Stages C/D optional — maintenance can run on any registry size.

## Task 7: maintenance-cron.yml GitHub Action

```yaml
name: Catalog maintenance
on:
  schedule:
    - cron: "0 3 * * *"   # nightly 3am UTC - link-health
    - cron: "0 4 * * 0"   # Sunday 4am - dedup-rescan + taxonomy-audit
    - cron: "0 5 1 */3 *" # quarterly 1st - citation-refresh
  workflow_dispatch:
    inputs:
      job:
        type: choice
        options: [link-health, dedup-rescan, taxonomy-audit, citation-refresh, shader-regression]

jobs:
  run:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - run: pip install -r scripts/research/requirements.txt
      - name: Run maintenance job
        run: python3 scripts/research/forge.py maintain "${{ github.event.schedule || github.event.inputs.job }}"
      - name: Open PR if fixes applied
        if: success() && hashFiles('reports/*.md') != ''
        uses: peter-evans/create-pull-request@v6
        with:
          commit-message: "chore(catalog): nightly maintenance fixes"
          branch: maintenance/auto-${{ github.run_id }}
          title: "Maintenance auto-PR: ${{ github.event.schedule }}"
          body-path: reports/latest-run.md
```

- [ ] **Step 1: Write workflow. Commit.**

## Risks

| Risk | Mitigation |
|------|-----------|
| Wayback Machine rate limits | Batch requests, exponential backoff, cache 24h |
| Shader regression false positives on Flutter SDK bump | Tolerate minor pixel-hash drift (Hamming distance ≤ 4); fail only on catastrophic difference |
| Maintenance PRs pile up if not merged | Add auto-close after 30 days; open next PR picks up where prior left off |
| Dashboard HTML grows too large at 10K entries | Paginate per-category or split per-tier HTML files |
| Cron failure invisible | Configure GitHub Actions email on failure; add badge to main README |
