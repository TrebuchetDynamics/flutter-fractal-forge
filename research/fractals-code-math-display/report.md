# Fractals of Many Kinds: math + display algorithms

## Method and limits

Retrieval used `rforge v0.1.3` across OpenAlex, Crossref, Semantic Scholar, and arXiv with six broad queries about escape-time rendering, iterated function systems, L-systems, strange attractors, GPU shaders, and procedural terrain/noise. The sweep returned 373 unique DOI-bearing records. No copyrighted full text was downloaded; this report uses bibliographic metadata/search results plus citation expansion metadata.

Semantic Scholar returned HTTP 429 for four query files, so coverage is weaker for that source. arXiv returned non-empty files for all six queries, but broad fractal terms also retrieve unrelated chaos/noise papers.

## Bottom line

For a fractal display/code project, the practical taxonomy is: escape-time complex dynamics, distance-estimated/raymarched implicit fractals, iterated-function-system attractors, grammar/L-system plants, chaotic/strange attractors, subdivision/replacement fractals, and procedural fractal noise/terrain. The core implementation split is whether the image is generated by per-pixel iteration, stochastic point iteration, turtle/grammar expansion, ODE integration, or height-field/noise synthesis.

Classic graphics papers found by the sweep anchor the display side: Hart et al., **“Ray tracing deterministic 3-D fractals”** (DOI `10.1145/74333.74363`, ACM SIGGRAPH/Computer Graphics proceedings) for ray tracing/distance-style 3D fractals; Musgrave et al., **“The synthesis and rendering of eroded fractal terrains”** (DOI `10.1145/74334.74337`, ACM SIGGRAPH/Computer Graphics proceedings) for terrain; and Prusinkiewicz et al., **“Visual models of plants interacting with their environment”** (DOI `10.1145/237170.237279`, ACM SIGGRAPH proceedings) plus **“Plant models faithful to botanical structure and development”** (DOI `10.1145/378456.378505`, ACM SIGGRAPH proceedings) for L-system/plant modeling.

## Main themes

### 1. Escape-time sets: Mandelbrot, Julia, Burning Ship, Multibrot, Newton basins

**Math.** Iterate a map and color by first escape time:

```text
z0 = pixel_to_complex(x, y)
zn+1 = f(zn, c)
escape when |zn| > R or n == maxIter
```

Common maps:

```text
Mandelbrot: z0 = 0,          zn+1 = zn^2 + c
Julia:      z0 = pixel,      zn+1 = zn^2 + k
Multibrot:  z0 = 0,          zn+1 = zn^p + c
BurningShip:z = (abs(Re z), abs(Im z)); zn+1 = z^2 + c
Newton:     zn+1 = zn - f(zn)/f'(zn); color by root reached
```

**Display.** Best fit for fragment shaders: one pixel = one independent orbit. Smooth coloring uses a continuous escape value:

```text
mu = n + 1 - log(log(|z|)) / log(power)
```

Relevant retrieved paper: Krawczyk and Krawczyk, **“Fractal art using variations on escape time algorithms in the complex plane”** (DOI `10.1080/17513470701210485`) directly targets escape-time art variants. Danca, **“A comparison of fractal dimension estimations for filled Julia fractal sets based on the escape time algorithm”** (DOI `10.1063/1.5095111`) shows the same iteration family used for quantitative analysis, not just images.

### 2. Distance-estimated 3D fractals: Mandelbulb, Mandelbox, Menger, Kaleidoscopic IFS

**Math.** Define an implicit/fractal distance estimator `DE(p)`, then sphere-trace along a ray:

```text
p = camera_origin
for step in 0..maxSteps:
    d = DE(p)
    if d < epsilon: hit
    p += ray_dir * d
```

Canonical families:

```text
Mandelbulb: spherical power map + c in R^3
Mandelbox: box fold + sphere fold + scale
Menger sponge: repeated abs/fold/scale carving
KIFS: repeated reflections across symmetry planes + scaling
```

**Display.** Needs camera rays, normals from finite differences, ambient occlusion/shadows from extra DE samples, and conservative step control to avoid holes. Retrieved anchor: Hart et al., **“Ray tracing deterministic 3-D fractals”** (DOI `10.1145/74333.74363`), with citation expansion producing 74 edges in `citation-graph-ray-tracing-deterministic-3d-fractals.json`.

### 3. Iterated Function Systems: Barnsley fern, Sierpinski, affine/tile attractors

**Math.** Choose affine maps with probabilities:

```text
p = (0, 0)
repeat N:
    choose i with probability wi
    p = Ai * p + bi
    plot(p)
```

Deterministic variant applies every transform to every point/shape each generation. Typical examples: Sierpinski triangle/carpet, Barnsley fern, Heighway dragon, Apollonian-like circle systems.

**Display.** CPU point-cloud accumulation works well; GPU compute or fragment ping-pong textures work for millions of points. Retrieved items include **“Automatic generation of iterated function systems”** (DOI `10.1016/0097-8493(94)90042-6`) and **“IFS Rendering Algorithms”** (DOI `10.1007/1-84628-686-7_5`).

### 4. L-systems and botanical fractals

**Math.** Rewrite symbols for `n` generations, then interpret with turtle graphics:

```text
axiom: F
rule:  F -> F[+F]F[-F]F
symbols: F draw, + turn, - turn, [ push state, ] pop state
```

Parametric/stochastic/context-sensitive L-systems add variables, random choices, and environment feedback.

**Display.** Expand grammar to line segments, meshes, or instanced branches/leaves. Retrieved anchors: Prusinkiewicz et al., **“Visual models of plants interacting with their environment”** (DOI `10.1145/237170.237279`) and **“Plant models faithful to botanical structure and development”** (DOI `10.1145/378456.378505`). L-Py is a modern framework retrieved by OpenAlex: Boudon et al., **“L-Py: An L-System Simulation Framework for Modeling Plant Architecture Development Based on a Dynamic Language”** (DOI `10.3389/fpls.2012.00076`).

### 5. Strange attractors and chaotic flows

**Math.** Integrate a dynamical system and plot the trajectory after burn-in:

```text
Lorenz:
dx/dt = sigma(y - x)
dy/dt = x(rho - z) - y
dz/dt = xy - beta z
```

Other useful families: Rössler, Aizawa, Thomas, Halvorsen, Chen, Clifford/Peter de Jong 2D maps.

**Display.** Use ODE integration (`Euler` for cheap previews, `RK4` for quality), color by speed/curvature/time, accumulate alpha into a framebuffer for density. Retrieved arXiv/Crossref items include **“A 3D Strange Attractor with a Distinctive Silhouette. The Butterfly Effect Revisited”** and **“Visualizing the Template of a Chaotic Attractor”** (DOI `10.1007/978-3-030-04414-5_8`).

### 6. Procedural fractal terrain and noise

**Math.** Fractional Brownian motion sums octaves of noise:

```text
fbm(p) = sum_{i=0}^{octaves-1} amplitude_i * noise(frequency_i * p)
amplitude_i = persistence^i
frequency_i = lacunarity^i
```

Variants: ridged multifractal, domain warping, erosion post-process, faulting, midpoint displacement, diamond-square.

**Display.** Height-field mesh, raymarched height map, or shader displacement. Retrieved anchor: Musgrave et al., **“The synthesis and rendering of eroded fractal terrains”** (DOI `10.1145/74334.74337`), with citation expansion producing 72 edges. Also retrieved: **“Procedural fractal terrains”** (DOI `10.1016/b978-155860848-1/50045-0`) and **“Terrain rendering in Frostbite using procedural shader splatting”** (DOI `10.1145/1281500.1281668`).

### 7. Replacement/subdivision fractals and tilings

**Math.** Repeatedly replace geometry with scaled copies: Koch curve/snowflake, Peano/Hilbert curves, dragon curve, Pythagoras tree, Penrose-like substitution tilings.

**Display.** Generate polylines/meshes recursively until projected segment size is below a pixel threshold. This family is simple and ideal for interactive educational controls because recursion depth and replacement rules are visible.

## Code-math implementation matrix

| Family | State | Core loop | Good renderer | Key controls |
|---|---|---|---|---|
| Mandelbrot/Julia | complex `z` | iterate map per pixel | fragment shader | center, zoom, maxIter, power, palette |
| Newton/basins | complex `z` | root-finding iteration | fragment shader | polynomial, tolerance, root colors |
| 3D DE fractals | 3D point/ray | sphere tracing | raymarch shader | camera, DE params, epsilon, AO |
| IFS | 2D/3D point | random affine transform | point cloud / accumulation texture | transforms, weights, samples |
| L-system | string/turtle state | rewrite + turtle draw | CPU mesh/line builder + GPU draw | axiom, rules, angle, depth |
| Strange attractor | ODE/map state | integrate trajectory | line/point accumulation | coefficients, timestep, burn-in |
| Terrain/noise | 2D sample point | octave noise sum | height mesh or shader | octaves, lacunarity, persistence, warp |
| Subdivision curves | geometry list | recursive replacement | polyline/mesh | rule, depth, angle/scale |

## Performance claims hygiene

Do not state headline speed/quality claims without exact paper, hardware, precision, sample count, resolution, and implementation conditions. Safe wording: “Hart et al. (DOI `10.1145/74333.74363`) is a retrieved anchor for ray tracing deterministic 3D fractals,” not “ray tracing fractals is fast.” For app benchmarks, measure shader compile status, frame time, step count, and pixel-difference stability locally.

## Evidence gaps

- The retrieval is broad, so many classic fractal sources may be books or older proceedings not well represented in DOI search.
- Semantic Scholar rate-limited four queries (`HTTP 429`), reducing CS citation ranking coverage.
- This report does not evaluate full-text algorithms line by line; it classifies implementable families from metadata and known math patterns.
- Visual acceptance still needs measurable criteria: pixel snapshots, histogram/palette checks, frame-time budgets, and screen-reader labels for controls.

## Implications for Flutter Fractal Forge

1. Treat each fractal as a module with a **math kernel**, **parameter schema**, **renderer type**, and **palette strategy**.
2. Keep fragment-shader modules for escape-time, Newton, distance-estimated 3D, and noise terrain.
3. Use CPU/generated buffers for L-systems, subdivision curves, and low-sample IFS; consider GPU accumulation for high-sample IFS/attractors.
4. Add presets by family: “Mandelbrot classic,” “Julia dendrite,” “Burning Ship,” “Mandelbox,” “Barnsley fern,” “Lorenz,” “Koch snowflake,” “ridged terrain.”
5. Validate visually with deterministic seeds and numerical image metrics, not subjective appearance.
